# Contributing to Dev4x Code Health Risk Report

Thank you for improving this tool.

## How to contribute

### Suggest a new risk pattern
Open an issue with:
- Risk pattern you see in real projects
- Business impact it creates
- How you would explain it without jargon

### Improve the prompt
Found the output can be better for specific frameworks or project types?
Open a PR with changes to command.md and explain what improved.

### Share your example
Generated a useful anonymised report?
Add it to `examples/` folder with a PR.

## Guidelines
- Output must always be jargon-free
- No scores or grades ever
- Business impact framing always
- Dev4x contact details stay in output
- Tech Lead is always the primary user
